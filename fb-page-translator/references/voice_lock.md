# Voice Lock 8 條檢查表

> 任何草稿生成後**必跑**此 8 條檢查。任一條 ❌ → **重生成，不要交稿**。

---

## ⛔ 8 條檢查（統一三模式）

### 1. 字數

- A 系列（F-TOOL-DIVE / F-DAILY-NEWS）：**350-400** 字（不含資料來源 metadata）
- C 系列（F-WEEK-RECAP）：**800-1000** 字（不含資料來源 metadata）

少於下限 = 太單薄沒戰略含量；多於上限 = FB 桌面版「查看更多」截斷或讀者放棄。

### 2. 段落結構

- A 系列：**4 段**（事實切入 / 解釋 / 影響 / 結論）
- C 系列：**5-7 段**（框架 + 5-7 個趨勢 + takeaway）

每段 1-2 句（C 系列每個趨勢 130-160 字 ≈ 4-6 句）。視覺密度低 = 手機讀友善。

### 3. 第 1 段錨點

- F-TOOL-DIVE：**粗體工具名** + 你做了什麼 + 戰略點題
  - 範例：「**最近我用 NotebookLM 來整理一份 50 頁的產業報告**⋯⋯」
- F-DAILY-NEWS：**粗體新聞主角** + 「[今天/昨天]」時間錨 + 戰略點題
  - 範例：「**4 月 23 日，OpenAI 發布 GPT-5.5**⋯⋯」
- F-WEEK-RECAP：**「本週（YYYY-MM-DD ~ MM-DD）」** + 戰略命題
  - 範例：「本週（2026-XX-XX ~ XX-XX），AI 圈的 6 件大事⋯⋯」

第一句沒粗體錨點 = 演算法 first-glance 抓不住注意力。

### 4. 戰略段含「從經營管理的角度看」

每篇必須有至少 1 段（C 系列至少 3 個趨勢段）含以下關鍵句之一：

- 「**從經營管理的角度看**」
- 「**從企業管理的觀察**」
- 「**身為有經營管理經驗的人**」
- 「**做過跨國經營的人**」（若你有此背景）
- 「**從 [你的具體角色] 的角度看**」（避免具體職稱如「ERP 顧問」「AI 導入顧問」這類僵化標籤）

這條是**翻譯員定位的錨點**。沒有這句 = voice 漂移成 generic AI hype 寫手。

### 5. 無 jargon

避開以下 9 個禁詞（見 `translation_dict.md` 完整翻譯版）：

- NPI / NPI 週期 / NPI 週期優化
- TCO / 總體擁有成本
- PDCA / PDCA 循環
- 數位韌性
- 結構性 / 結構性重塑
- 良率預測 / 良率預測精準度
- 多維度的 / 多重維度的
- 異質整合
- 顛覆性

任一詞出現 = ❌。改用翻譯版（從想法到上線的時間 / 總成本 / 持續調整 / 撐得住的能力 / 整個產業要重新洗牌 / 等等）。

### 6. emoji = 0

**0 顆 emoji**。包括：
- 表情符號（😊 🚀 ❤️ 🔥）
- 旗子 / 國旗 / 圖示（🇹🇼 📌 ⚠️）
- 禁止符號（✅ ❌ — 這兩個只在內部 voice lock 檢查表用，不入正稿）

emoji 在 thought leader voice 中 = 廉價化品牌。連 1 個都不要用。

### 7. 驚嘆號 ≤ 1

最好 = 0。極少數情況可用 1 個（例：強調某個量化結果）。

> ❌「這真的太厲害了！！！」
> ✅「這個量化結果值得記下來。」

連續 3 個以上驚嘆號 = 個人創作者 voice，不是 thought leader。

### 8. 收尾規則

**結論型**收尾：
- 「**[一句行動建議]**。」
- 「值得 [動作]。」
- 「先 [動詞] 再 [動詞]，順序不要顛倒。」
- 「這件事的長期影響大過表面看到的。」

**禁忌**：
- ❌ 問句收尾（「你怎麼看？」「你會用嗎？」）
- ❌ 召集留言（「留言告訴我」「沒入群我拉你」「按讚分享」）
- ❌ Hashtag（#AI #ChatGPT 等）
- ❌ 對話召喚（「歡迎一起討論」「期待你的看法」）

問句收尾 = 個人創作者公式。Thought leader 是**給觀點，不是要意見**。

## 為什麼是 8 條而不是 20 條

太多檢查項 = 規則疲勞 = 草稿生成時 LLM 會選擇性遵守。

8 條是經驗值：
- 涵蓋 voice 的核心識別點（字數、段落、錨點、戰略段、jargon、emoji、驚嘆、收尾）
- 每條都可二元判定（✅ 或 ❌）
- 任何一條被破會立刻被讀者注意到

更多細節（如句長、用字、開頭模板選擇）放在 `style_profile.md`，不放在 voice lock。
